# NatSpec 格式

**NatSpec (Ethereum Natural Language Specification Format)**，即以太坊自然语言规范格式。它是 Solidity 官方推荐的文档注释标准。

如果你写过 Java，它非常像 Javadoc；如果你写过 C++，它深受 Doxygen 的启发。

#### 标签 (Tags) 详解

| **标签**          | **上下文**   | **用途**                    |
| --------------- | --------- | ------------------------- |
| **@title**      | 合约、库、接口   | 描述合约/接口的标题                |
| **@author**     | 同上        | 作者姓名                      |
| **@notice**     | 几乎所有地方    | **给终端用户的通知**，解释“这东西是做什么的” |
| **@dev**        | 几乎所有地方    | **给开发者的详情**，解释实现细节或背景     |
| **@param**      | 函数、事件     | 文档化参数（需后跟参数名）             |
| **@return**     | 函数、公共状态变量 | 文档化返回值                    |
| **@inheritdoc** | 函数        | 从指定的基类合约中拷贝缺失的标签          |
| **@custom:...** | 任何地方      | 自定义标签，用于第三方工具分析（如安全审计）    |
